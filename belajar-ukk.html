🐘 PHP Native

PHP & Database

PHP berjalan di server — merakit data sebelum dikirim ke browser.

01

Koneksi ke Database

▾

Baris pertama PHP adalah membuka jalur komunikasi ke MySQL. Variabel $db menyimpan jalur ini dan dipakai di seluruh file.

PHP 
$db = mysqli_connect('localhost', 'root', '', 'lat_todo')
     or die("Koneksi ke database gagal.");
Parameter Nilai Artinya
'localhost' server Database di komputer yang sama
'root' username Username default XAMPP
'' password Kosong — default XAMPP tanpa password
'lat_todo' database Nama database yang digunakan
⚠️

or die() — Kalau koneksi gagal, PHP langsung berhenti dan tampilkan pesan. Tanpa ini, error muncul acak di seluruh halaman.

02

Tambah Tugas — Form POST

▾

Saat tombol "Simpan" ditekan, browser mengirim data form via POST. PHP menangkap, validasi, lalu simpan ke database.

PHP 
if (isset($_POST['submit'])) {
  $task = mysqli_real_escape_string($db, $_POST['task_input']);
  $tgl  = $_POST['tanggal'];
  $prio = $_POST['priority'];

  if (!empty($task) && !empty($tgl) && !empty($prio)) {
    mysqli_query($db, "INSERT INTO tasks (task, tanggal, priority)
      VALUES ('$task', '$tgl', '$prio')");
    header('location:index.php?notif=added');
    exit();
  }
}
Kode Fungsinya
isset($_POST['submit']) Cek apakah tombol submit ditekan
mysqli_real_escape_string() Anti SQL Injection — bersihkan karakter berbahaya
!empty() "Tidak kosong" — validasi semua field
INSERT INTO tasks Tambah baris baru ke tabel database
header('location:...') Redirect ke halaman lain setelah berhasil
exit() Hentikan PHP — wajib setelah header()
💡

Kenapa exit() wajib setelah header()? header() hanya mendaftarkan redirect, tapi PHP tetap lanjut ke bawah. exit() memaksa PHP berhenti total seketika.

03

Hapus Tugas — Link GET

▾

Berbeda dengan POST, hapus tugas pakai GET — ID dikirim langsung di URL yang terlihat.

PHP 
// URL: index.php?del_task=5
if (isset($_GET['del_task'])) {
  $id = (int) $_GET['del_task'];
  mysqli_query($db, "DELETE FROM tasks WHERE id=$id");
  header('location:index.php?notif=deleted');
  exit();
}
Kode Fungsinya
$_GET['del_task'] Ambil nilai dari URL: ?del_task=5
(int) Type casting — paksa jadi angka, cegah SQL Injection
DELETE FROM tasks WHERE id= Hapus baris dengan ID tertentu dari database
🔐

POST vs GET: POST = data di body (tidak terlihat di URL). GET = data di URL (terlihat). Pakai POST untuk data sensitif, GET untuk navigasi/hapus sederhana.

04

Update Status via AJAX (PHP menerima)

▾

Bagian ini menerima kiriman dari JavaScript tanpa reload halaman. PHP update database lalu exit — tidak kirim HTML balik.

PHP 
if (isset($_POST['action']) && $_POST['action'] == 'update_status') {
  $id     = (int) $_POST['id'];
  $status = mysqli_real_escape_string($db, $_POST['status']);
  mysqli_query($db, "UPDATE tasks SET status='$status' WHERE id=$id");
  exit(); // Jangan kirim HTML ke JavaScript!
}
🔗

Dipanggil oleh JavaScript via fetch('index.php', { method:'POST' }). exit() penting agar PHP tidak ikut mengirim halaman HTML penuh yang tidak dibutuhkan JS.

05

Ambil Data & Loop (while + foreach)

▾
PHP 
$tasks = [];
$result = mysqli_query($db, "SELECT * FROM tasks ORDER BY id DESC");
while ($row = mysqli_fetch_array($result)) {
  $row['status'] = $row['status'] ?: 'belum'; // Elvis operator
  $tasks[] = $row;
}
Kode Fungsinya
SELECT * FROM tasks ORDER BY id DESC Ambil semua data, terbaru di atas
while ($row = mysqli_fetch_array()) Ambil baris satu per satu sampai habis
$row['status'] ?: 'belum' Elvis: kalau kosong → isi 'belum'
$tasks[] = $row Tambah ke array tanpa set index manual
06

Operator ?? dan ?: (Null Coalescing & Elvis)

▾
PHP 
// Null Coalescing (??) — kalau tidak ada, pakai default
$notif = $_GET['notif'] ?? '';
// Kalau $_GET['notif'] ada → pakai itu. Tidak ada → pakai ''

// Elvis (?:) — kalau falsy, pakai default
$row['status'] = $row['status'] ?: 'belum';
// Kalau status tidak kosong → pakai itu. Kosong → 'belum'
💡

Bedanya: ?? khusus cek null/tidak ada. ?: cek "falsy" (null, false, 0, string kosong). Keduanya pengganti if-else dalam satu baris.

07

Ternary Operator & array_filter

▾
PHP 
// Ternary: kondisi ? kalau_benar : kalau_salah
$class = $item['priority'] == 'High'
  ? 'danger'
  : ($item['priority'] == 'Medium' ? 'warning' : 'success');

// array_filter — saring array
$filtered = array_filter($tasks,
  fn($t) => strtolower($t['status']) == $slug
);
Kode Fungsinya
? 'danger' : 'success' If-else singkat dalam satu baris
array_filter($arr, fn) Saring array, ambil yang sesuai kondisi
fn($t) => Arrow function — fungsi singkat PHP 7.4+
strtolower() Huruf kecil semua — biar tidak case-sensitive
08

Fungsi alert() — Helper Notifikasi

▾

PHP bisa punya fungsi buatan sendiri (user-defined function). Fungsi alert() ini menghasilkan HTML Bootstrap Alert setiap dipanggil — satu fungsi dipakai berkali-kali untuk berbagai jenis notifikasi.

PHP 
function alert($type, $icon, $txt)
{
    return "
    <div class='alert alert-$type alert-dismissible fade show shadow-sm text-center'>
        <i class='fa-solid fa-$icon me-2'></i> $txt
        <button type='button' class='btn-close' data-bs-dismiss='alert'></button>
    </div>";
}

// Contoh pemanggilan:
$msg = alert('success', 'circle-check', 'Tugas berhasil ditambahkan.');
// Hasilnya: <div class='alert alert-success ...'>

$msg = alert('danger', 'triangle-exclamation', 'Harap isi semua kolom.');
// Hasilnya: <div class='alert alert-danger ...'>
Bagian KodeArti
function alert($type, $icon, $txt)Definisi fungsi dengan 3 PARAMETER — nilai yang dikirim saat dipanggil
return "..."Fungsi menghasilkan string HTML. Tidak langsung tampil — harus di-echo atau disimpan ke variabel
alert-$typeString interpolation — PHP sisipkan nilai $type ke dalam string. alert-success, alert-danger, dll.
fa-$iconNama icon Font Awesome disisipkan dinamis — circle-check, trash-can, triangle-exclamation
data-bs-dismiss="alert"Bootstrap JS otomatis sembunyikan alert saat tombol ✕ diklik
💡

String Interpolation PHP: Variabel di dalam tanda kutip ganda (") otomatis digantikan nilainya. "alert-$type" dengan $type='success' → "alert-success". Ini tidak berlaku di tanda kutip tunggal (').

♻️

Prinsip DRY (Don't Repeat Yourself): Tanpa fungsi ini, kita harus tulis HTML alert yang panjang setiap kali butuh notifikasi. Dengan fungsi, cukup panggil sekali dengan parameter berbeda. Kode makin rapi dan mudah diubah.

09

Notifikasi dari URL — Pola Redirect + Notif

▾

Ini pola klasik PHP: setelah operasi berhasil, redirect ke URL dengan parameter notif, lalu baca parameter itu untuk tampilkan pesan. Tanpa ini, kalau halaman di-refresh ulang, form akan tersubmit lagi.

PHP — 3 Langkah Alur Notifikasi 
// ① LANGKAH 1: Setelah berhasil, redirect dengan parameter notif
mysqli_query($db, "INSERT INTO tasks ...");
header('location:index.php?notif=added');   // URL: ...?notif=added
exit();

mysqli_query($db, "DELETE FROM tasks WHERE id=$id");
header('location:index.php?notif=deleted'); // URL: ...?notif=deleted
exit();

// ② LANGKAH 2: Baca parameter URL (halaman sudah reload bersih)
$notif = $_GET['notif'] ?? '';
// ??: kalau 'notif' tidak ada di URL → pakai string kosong

// ③ LANGKAH 3: Cocokkan nilai notif → tampilkan pesan
if ($notif == 'added')
    $msg = alert('success', 'circle-check', 'Tugas berhasil ditambahkan.');
if ($notif == 'deleted')
    $msg = alert('warning', 'trash-can', 'Tugas berhasil dihapus.');

// ④ LANGKAH 4: Cetak $msg di HTML
<?= $msg ?>
KodeFungsinya
$_GET['notif'] ?? ''Ambil nilai ?notif dari URL. Null coalescing: kalau tidak ada → string kosong (tidak error)
?? vs isset()$_GET['x'] ?? '' lebih ringkas dari isset($_GET['x']) ? $_GET['x'] : '' — hasilnya sama
alert('success', 'circle-check', '...')Panggil fungsi helper dari Topik 08 — hasilkan HTML alert hijau
alert('warning', 'trash-can', '...')Panggil fungsi helper — hasilkan HTML alert kuning
<?= $msg ?>Shorthand echo. Cetak isi HTML dari variabel $msg ke halaman
⚠️

Kenapa harus redirect dulu? Kalau kita langsung tampilkan notifikasi tanpa redirect, user yang refresh halaman akan memunculkan dialog "Kirim ulang form?". Pola redirect ini (disebut PRG: Post-Redirect-Get) mencegah hal itu.

10

Array Asosiatif Kolom Kanban — Struktur & foreach

▾

Semua konfigurasi kolom disimpan dalam satu array asosiatif. Ini prinsip DRY — daripada tulis 3 section HTML yang hampir sama, cukup satu foreach.

PHP 
$columns = [
  'belum'   => ['Belum Dikerjakan', 'list-ul',     'text-secondary', 'bg-white text-dark',    ''],
  'proses'  => ['Proses',          'spinner',      'text-primary',   'bg-white text-primary', '#e3f2fd'],
  'selesai' => ['Selesai',         'circle-check', 'text-success',   'bg-white text-success', '#e8f5e9']
];
//  key        [0]judul          [1]icon         [2]warna-judul    [3]warna-badge         [4]bg-kolom

foreach ($columns as $slug => $meta):
  // $slug = 'belum' / 'proses' / 'selesai'
  // $meta = array berisi konfigurasi kolom itu
  $filtered_tasks = array_filter($tasks,
    fn($t) => strtolower($t['status']) == $slug
  );
endforeach;
Index $metaNilainya (contoh kolom 'belum')Dipakai untuk
$meta[0]'Belum Dikerjakan'Judul yang tampil di header kolom
$meta[1]'list-ul'Nama icon Font Awesome: fa-list-ul
$meta[2]'text-secondary'Class Bootstrap untuk warna teks judul kolom
$meta[3]'bg-white text-dark'Class Bootstrap untuk badge jumlah tugas
$meta[4]'' (kosong = putih)Warna hex background kolom via inline style. Kosong = #fff
💡

foreach pakai titik dua (:) bukan kurung kurawal? Ini alternative syntax PHP, lebih rapi saat ada campuran PHP + HTML. foreach(...): ... endforeach; vs foreach(...) { ... } — keduanya valid tapi yang titik dua lebih mudah dibaca di template HTML.

🎯

Array asosiatif (key → value) berbeda dari array biasa (0,1,2,3). Key bisa berupa string: $columns['belum'] lebih deskriptif dari $columns[0]. Di foreach, kita gunakan $slug => $meta untuk dapat key dan value sekaligus.

11

Render Kartu Tugas — htmlspecialchars, date(), data-id

▾

Setiap kartu dicetak menggunakan data dari database. Ada 3 hal penting: keamanan XSS, format tanggal, dan atribut data-* untuk JavaScript.

PHP — Logika Sebelum Render 
foreach ($filtered_tasks as $item):
  // Tentukan class warna badge prioritas (ternary bersarang)
  $priority_class = $item['priority'] == 'High'
    ? 'danger'    // High → merah
    : ($item['priority'] == 'Medium'
        ? 'warning'   // Medium → kuning
        : 'success'); // Low → hijau

  // Variabel boolean: apakah kartu ini di kolom selesai?
  $is_finished = ($slug == 'selesai'); // true atau false
endforeach;
PHP dalam HTML — 5 Teknik Penting 
<!-- ① Atribut draggable + data-id untuk JavaScript -->
<div draggable="true" data-id="<?= $item['id'] ?>">

<!-- ② Style + class kondisional berdasarkan $is_finished -->
<div class="card <?= $is_finished ? 'text-muted' : '' ?>"
     style="<?= $is_finished ? 'opacity:0.75;' : '' ?>">

<!-- ③ Badge prioritas — class warna dari $priority_class -->
<span class="badge text-bg-<?= $priority_class ?>"><?= $item['priority'] ?></span>

<!-- ④ XSS Protection — htmlspecialchars wajib untuk output user input -->
<p class="card-text <?= $is_finished ? 'text-decoration-line-through' : 'text-dark' ?>">
  <?= htmlspecialchars($item['task']) ?>
</p>

<!-- ⑤ Format tanggal: dari '2026-04-16' → '16 Apr 2026' -->
<?= date('d M Y', strtotime($item['tanggal'])) ?>
KodeFungsinya
draggable="true"Atribut HTML5 — izinkan browser untuk drag elemen ini. Wajib ada agar drag-drop bisa berjalan
data-id="<?= $item['id'] ?>"Simpan ID tugas dari database ke dalam HTML. JavaScript baca ini saat dragend untuk tahu ID yang harus diupdate
htmlspecialchars($item['task'])Ubah < > & " dari input user menjadi entitas HTML yang aman. Mencegah serangan XSS
strtotime($item['tanggal'])Ubah string tanggal MySQL ('2026-04-16') menjadi Unix timestamp — angka detik sejak 1 Jan 1970
date('d M Y', timestamp)Format timestamp: d=tanggal, M=bulan singkat (Apr), Y=tahun 4 digit. Hasil: '16 Apr 2026'
$is_finished ? 'opacity:0.75' : ''Inline ternary: kalau selesai → set opacity 75%, kalau belum → tidak ada style tambahan
🛡️

XSS Attack (Cross-Site Scripting): Kalau user input <script>alert('hacked')</script> sebagai nama tugas, tanpa htmlspecialchars() script itu akan dieksekusi browser. htmlspecialchars() mengubah karakter berbahaya menjadi teks biasa: &lt;script&gt; yang aman.

🔗

Hubungan data-id dengan JavaScript: PHP mencetak data-id="5" di HTML. Saat kartu di-drag, JS baca dengan draggable.getAttribute('data-id') → dapat angka 5 → kirim ke PHP lewat fetch untuk UPDATE database.

JavaScript & AJAX

JavaScript berjalan di browser — membuat halaman interaktif tanpa reload.

01

querySelector & forEach

▾
JS 
const draggables = document.querySelectorAll('.draggable');
const droppables = document.querySelectorAll('.droppable');

draggables.forEach(draggable => {
  // lakukan sesuatu untuk setiap kartu
});
Kode Fungsinya
const Deklarasi variabel yang nilainya tidak bisa diubah
querySelectorAll('.cls') Ambil SEMUA elemen dengan class itu (NodeList)
querySelector('.cls') Ambil SATU elemen pertama yang cocok
.forEach(item => {}) Loop setiap elemen. item = elemen saat ini
02

Event Listener & classList

▾
JS 
draggable.addEventListener('dragstart', () => {
  draggable.classList.add('dragging');    // tambah class CSS
});

draggable.addEventListener('dragend', () => {
  draggable.classList.remove('dragging'); // hapus class CSS
});
Event Kapan terjadi
dragstart User mulai menarik elemen
dragend User melepas elemen (di mana saja)
dragover Elemen melayang di atas zona drop
dragleave Elemen meninggalkan zona drop
drop Elemen dilepas tepat di zona drop
🎨

classList.add/remove adalah jembatan JS ↔ CSS. JS tambah class → CSS aktifkan efek visual. JS hapus class → efek hilang. CSS sendiri tidak bisa "memutuskan" kapan aktif.

03

preventDefault & appendChild

▾
JS 
droppable.addEventListener('dragover', e => {
  e.preventDefault(); // ← WAJIB! Izinkan drop
  droppable.classList.add('drag-over');

  const item = document.querySelector('.dragging');
  if (item) droppable.appendChild(item);
});
⚠️

e.preventDefault() WAJIB di dragover! Browser secara default melarang drop di sembarang tempat. Tanpa ini, event drop tidak akan pernah terjadi sama sekali.

💡

appendChild(item) memindahkan elemen HTML yang sedang ditarik menjadi anak dari kolom tujuan. Inilah yang membuat kartu "pindah kolom" secara visual di layar.

04

getAttribute & data-* Attribute

▾
HTML + JS 
<!-- HTML simpan data di atribut data-* -->
<div class="card draggable" draggable="true" data-id="5">
<div class="droppable" data-status="proses">

// JS baca atributnya
const id        = draggable.getAttribute('data-id');         // "5"
const newStatus = draggable.parentElement.getAttribute('data-status'); // "proses"
Kode Fungsinya
getAttribute('data-id') Baca nilai atribut data-id dari elemen HTML
draggable.parentElement Akses elemen INDUK kartu — yaitu kolom tempat mendarat
05

Fetch API — AJAX tanpa Reload

▾

Fetch adalah cara JS kirim data ke server tanpa reload halaman. Inti dari fitur drag & drop yang menyimpan ke database.

JS 
const formData = new FormData();
formData.append('action', 'update_status');
formData.append('id', id);
formData.append('status', newStatus);

fetch('index.php', { method: 'POST', body: formData })
  .then(res => res.text())
  .then(() => { /* update tampilan */ })
  .catch(err => console.error('Error:', err));
Kode Fungsinya
new FormData() Buat paket data kosong (seperti mengisi form HTML)
formData.append(key, val) Isi paket data dengan satu field
fetch(url, { method, body }) Kirim request ke server
.then() "Kalau berhasil, lakukan ini" — callback sukses
.catch() Tangkap error kalau request gagal

⛓️ Alur Lengkap: Drag Kartu ke Kolom Lain

1
HTML merender kartu
PHP mencetak draggable="true" dan data-id="5" pada setiap kartu tugas.
2
JavaScript pasang event listener dragstart
Saat kartu mulai ditarik, JS tambah class .dragging via classList.add().
3
CSS bereaksi ke class .dragging
Kartu jadi transparan (opacity 0.5) dan sedikit membesar (scale 1.02) — sinyal visual sedang ditarik.
4
JavaScript event dragover — e.preventDefault()
Browser secara default melarang drop. preventDefault() membatalkan aturan itu. Plus tambah class .drag-over ke kolom tujuan.
5
CSS bereaksi ke class .drag-over
Background kolom tujuan berubah warna — sinyal "kamu bisa drop di sini".
6
JavaScript — appendChild() pindahkan kartu
Kartu jadi anak dari kolom tujuan. Secara visual kartu sudah berpindah kolom di layar.
7
JavaScript event dragend — bersih + ambil data
Remove class .dragging. Ambil data-id dari kartu dan data-status dari kolom tujuan.
8
JavaScript fetch() — kirim data ke PHP tanpa reload
FormData berisi action='update_status', id, dan status baru dikirim via POST ke index.php.
9
PHP menerima → UPDATE database → exit()
PHP cek action='update_status', jalankan UPDATE tasks SET status='...', lalu exit() agar tidak kirim HTML balik.
10
JavaScript .then() — update tampilan kartu
Kalau status='selesai': opacity 0.75 + text-muted + text-decoration-line-through. Semua tanpa reload.
11
JavaScript updateColumnCounts() — update badge angka
Loop semua kolom, hitung jumlah .draggable di dalamnya, update teks badge dengan angka baru.

🔥 Paling Umum

Jika tanggal sudah lewat, apakah status otomatis pindah ke Selesai?

▾
🎓

Skenario Guru: "Lihat — tugas ini tanggalnya kemarin tapi statusnya masih 'Belum Dikerjakan'. Apa yang terjadi di sistemmu? Apakah PHP otomatis memindahkannya ke Selesai?"

Jawaban Benar: TIDAK. Status tidak pernah berubah otomatis di kode ini. Status hanya berubah saat user drag kartu secara manual. PHP tidak punya mekanisme auto-check saat halaman dibuka.

💡

Kenapa tidak otomatis? PHP hanya berjalan saat ada HTTP request dari browser. Tidak ada background task, timer, atau cron job. Untuk auto-move, kamu perlu tambah logic PHP saat halaman dibuka.

Solusi: Tambah deteksi overdue saat loop data

PHP — Tambah di dalam while loop (ambil data) 
while ($row = mysqli_fetch_array($tasks_query)) {
    $row['status'] = $row['status'] ?: 'belum';

    // TAMBAHAN: Deteksi dan auto-update ke 'terlambat' jika overdue
    if ($row['status'] == 'belum'
        && strtotime($row['tanggal']) < strtotime('today')) {

        mysqli_query($db,
            "UPDATE tasks SET status='terlambat' WHERE id={$row['id']}"
        );
        $row['status'] = 'terlambat'; // Update array agar tampilan ikut berubah
    }

    $tasks[] = $row;
}
📌

strtotime('today') menghasilkan Unix timestamp awal hari ini (00:00:00). Membandingkan dengan timestamp tanggal tugas: kalau tanggal tugas lebih kecil → sudah lewat. Kalau pakai 'selesai' bukan 'terlambat' juga bisa, tergantung kebutuhan aplikasi.

🔥 Paling Umum

Coba ketik <script>alert('hacked')</script> sebagai nama tugas!

▾
🎓

Skenario Guru: "Sekarang coba ketikkan <script>alert('XSS')</script> di kolom nama tugas, lalu simpan. Apa yang terjadi di browsermu?"

Jawaban: Di kode UKK ini — AMAN. Karena saat menampilkan tugas ada htmlspecialchars(). Tapi perhatikan: input tersimpan apa adanya ke database, yang berubah hanya saat DITAMPILKAN.

PHP — Perbedaan Ada/Tanpa htmlspecialchars 
// Input user: <script>alert('hacked')</script>

// TANPA htmlspecialchars — BERBAHAYA! ❌
<?= $item['task'] ?>
// Output ke HTML: <script>alert('hacked')</script>
// Browser MENJALANKAN script ini → XSS Attack berhasil!

// DENGAN htmlspecialchars — AMAN ✔
<?= htmlspecialchars($item['task']) ?>
// Output ke HTML: &lt;script&gt;alert('hacked')&lt;/script&gt;
// Browser menampilkan sebagai TEKS biasa, tidak dieksekusi

// Karakter yang dikonversi htmlspecialchars():
// < → &lt;    > → &gt;    & → &amp;    " → &quot;
🛡️

Dua serangan berbeda: mysqli_real_escape_string melindungi dari SQL Injection (serangan di database). htmlspecialchars melindungi dari XSS (serangan di browser user). Keduanya diperlukan untuk keamanan lengkap!

⚡ Sedang

Coba hapus e.preventDefault() di dragover. Apa yang terjadi?

▾
🎓

Skenario Guru: "Di script.js ada baris e.preventDefault(). Coba hapus baris itu saja, jangan yang lain. Lalu coba drag kartu. Apa yang berubah?"

Jawaban: Drag & drop BERHENTI TOTAL. Kartu tidak bisa dipindah ke kolom lain sama sekali.

JavaScript — Penjelasan Mengapa WAJIB Ada 
droppable.addEventListener('dragover', e => {
    e.preventDefault(); // ← HAPUS INI = drag&drop mati total

    // KENAPA WAJIB?
    // Browser punya aturan default:
    // "Elemen HTML TIDAK boleh menerima drop."
    //
    // e.preventDefault() membatalkan aturan default itu.
    //
    // Tanpa baris ini:
    // - Browser menolak drop sebelum event 'drop' terjadi
    // - Event 'drop' tidak akan pernah terpanggil
    // - appendChild() tidak pernah berjalan
    // - Kartu tidak bisa pindah kolom
});
🎯

Aturan emas HTML5 Drag & Drop: Selalu panggil e.preventDefault() di listener dragover. Ini satu-satunya cara mengizinkan drop di elemen HTML. Ini bukan bug — ini desain standar W3C untuk drag & drop.

Internet mati SAAT drag kartu — apa yang terjadi?

▾
🎓

Skenario Guru: "User drag kartu ke kolom 'Selesai'. Kartu sudah pindah di layar. Tapi tiba-tiba internet mati sebelum fetch() selesai. Apa yang terjadi?"

Jawaban: Visual berubah, database TIDAK terupdate. Kartu pindah di layar karena appendChild() sudah berjalan, tapi fetch() gagal → .catch() terpanggil → PHP tidak pernah terima request → database tidak berubah.

JavaScript — Solusi error handling yang lebih baik 
fetch('index.php', { method: 'POST', body: formData })
  .then(res => res.text())
  .then(() => {
    updateColumnCounts();  // Update badge jumlah
  })
  .catch(err => {
    console.error('Gagal update:', err);

    // PERBAIKAN: Beri tahu user dan reload halaman
    alert('Gagal menyimpan. Periksa koneksi internet!');
    window.location.reload(); // Reset visual sesuai database
  });
💡

Ini contoh inkonsistensi state: tampilan (visual) dan data (database) tidak sinkron. Solusi terbaik: kalau .catch() terpanggil, reload halaman agar tampilan kembali sesuai data yang benar-benar tersimpan di database.

✅ Mudah

Tampilkan tugas High priority selalu di atas dalam setiap kolom!

▾
🎓

Skenario Guru: "Tugas prioritas High kan lebih penting. Bisa ubah kode agar High selalu muncul paling atas di kolom, lalu Medium, lalu Low?"

Solusi: Ubah query SQL dengan ORDER BY CASE WHEN untuk urutan custom.

PHP — Ubah query SELECT 
// SEBELUM: Hanya urut berdasarkan id (terbaru dulu)
$tasks_query = mysqli_query($db,
    "SELECT * FROM tasks ORDER BY id DESC"
);

// SESUDAH: Urut priority dulu, lalu id
$tasks_query = mysqli_query($db,
    "SELECT * FROM tasks
     ORDER BY
       CASE priority
         WHEN 'High'   THEN 1
         WHEN 'Medium' THEN 2
         WHEN 'Low'    THEN 3
         ELSE 4
       END ASC,
       id DESC"
);
💡

CASE WHEN di SQL seperti switch-case — beri angka ke setiap nilai. High=1, Medium=2, Low=3. ORDER BY angka itu ASC (kecil ke besar): High (angka 1) muncul duluan. ini adalah custom sort order di SQL.

🔴 Sulit

Dua user drag kartu yang sama bersamaan — siapa yang menang?

▾
🎓

Skenario Guru: "Kamu dan temanmu sama-sama buka halaman ini. Kamu drag kartu yang sama ke 'Selesai', temanmu drag kartu yang sama ke 'Proses', di waktu yang bersamaan. Apa yang terjadi?"

Jawaban: “Last Write Wins.” Siapa yang terakhir mengirim fetch() POST ke server — itulah yang tersimpan di database. Visual masing-masing browser bisa tidak sinkron dengan database.

Ilustrasi Race Condition 
// User A drag ke "selesai" → fetch POST → DB: status='selesai'
// User B drag ke "proses"  → fetch POST → DB: status='proses'
//
// Jika request B tiba SETELAH A:
//   Database akhir: status='proses' (B menang)
//   Visual A: kartu di kolom 'selesai' ← TIDAK SINKRON dengan DB!
//   Visual B: kartu di kolom 'proses'  ← sesuai DB
//
// Ini disebut RACE CONDITION — kode UKK tidak punya locking

// Solusi tingkat lanjut (informasi tambahan):
// Tambah kolom 'updated_at TIMESTAMP' di tabel tasks
// Saat UPDATE, cek dulu apakah updated_at masih sama:
"UPDATE tasks SET status='$status', updated_at=NOW()
 WHERE id=$id AND updated_at='$last_known_timestamp'"
// Jika 0 rows affected = orang lain sudah mengubah duluan
💡

Race Condition adalah masalah umum dalam sistem multi-user. Solusi sederhana: reload halaman setelah drag berhasil (tapi performa turun). Solusi profesional: optimistic locking dengan timestamp/version number di database.
